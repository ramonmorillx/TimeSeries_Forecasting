El modelo K-Nearest Neighbors (KNN) es un algoritmo de aprendizaje supervisado utilizado tanto para problemas de clasificacin como de regresin. Se basa en la idea de que las instancias de datos similares tienden a estar cercanas entre s en el espacio de caractersticas. En el contexto de regresin, KNN predice el valor de una variable objetivo en funcin de los valores de los k vecinos ms cercanos en el conjunto de entrenamiento. La distancia entre puntos se calcula tpicamente mediante mtricas como la Euclidiana, y el valor predicho se obtiene tomando la media (en regresin) o la moda (en clasificacin) de los vecinos ms cercanos. Este modelo es sencillo, intuitivo y no paramtrico, lo que significa que no hace suposiciones sobre la distribucin de los datos. Sin embargo, su rendimiento puede verse afectado por la eleccin de k y la escala de las caractersticas, y tiende a ser computacionalmente costoso a medida que aumenta el tamao del conjunto de datos.

Modelo KNN

El modelo K-Nearest Neighbors (KNN) puede ser utilizado para predecir las series temporales, en nuestro caso, pasajeros de autobs y metro de Madrid al identificar los k perodos ms similares a un mes dado en funcin de las caractersticas histricas, como el nmero de pasajeros en meses anteriores. A travs de una medida de distancia, como la distancia Euclidiana, el modelo encuentra los k puntos temporales ms cercanos y utiliza las predicciones de estos vecinos para estimar el valor del mes futuro. En el contexto de las series temporales, KNN puede ser til para capturar patrones similares en los datos pasados, pero su rendimiento depender de la eleccin adecuada de k y de la normalizacin de los datos. Sin embargo, puede resultar costoso en trminos computacionales cuando se manejan grandes cantidades de datos histricos.

# Definir una funcin para preparar los datos
def prepare_data(df, n_lags):
    X, y = [], []
    for i in range(n_lags, len(df)):
        X.append(df[i-n_lags:i])
        y.append(df[i])
    return np.array(X), np.array(y)

# Definir una funcin para entrenar y predecir con KNN
def knn_predict(df, n_lags):
    X, y = prepare_data(df.values, n_lags)
    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.1, shuffle=False)
    
    # Aplicar GridSearchCV para encontrar los mejores hiperparmetros
    param_grid = {'n_neighbors': range(1, 21)}
    grid_search = GridSearchCV(KNeighborsRegressor(), param_grid, cv=5, scoring='r2')
    grid_search.fit(X_train, y_train)
    
    best_model = grid_search.best_estimator_
    y_pred = best_model.predict(X_test)
    
    return X_train, X_test, y_train, y_test, y_pred, best_model

# Definir una funcin para graficar los resultados
def plot_results(df, y_train, y_test, y_pred, title):
    plt.figure(figsize=(14, 5))
    plt.plot(df.index[-len(y_test):], y_test, label='Test')
    plt.plot(df.index[-len(y_test):], y_pred, label='Prediction')
    plt.plot(df.index[:len(y_train)], y_train, label='Train')
    plt.title(title)
    plt.legend()
    plt.show()

# Aplicar el modelo y graficar los resultados para cada dataframe mensual
titles = ['Autobs Mensual', 'Metro Mensual', 'Combinado Mensual']
for df, title in zip([df_autobus_month, df_metro_month, df_combined_month], titles):
    X_train, X_test, y_train, y_test, y_pred, best_model = knn_predict(df, n_lags=12)
    plot_results(df, y_train, y_test, y_pred, title)
    print(f"Best model parameters for {title}: {best_model.get_params()}")

# Definir una funcin para calcular MSE, RMSE y MAPE
def calculate_metrics(y_test, y_pred):
    mse = mean_squared_error(y_test, y_pred)
    rmse = np.sqrt(mse)
    mape = mean_absolute_percentage_error(y_test, y_pred)
    return mse, rmse, mape

# Calcular y mostrar las mtricas para cada conjunto de datos
for df, title in zip([df_autobus_month, df_metro_month, df_combined_month], titles):
    X_train, X_test, y_train, y_test, y_pred, best_model = knn_predict(df, n_lags=12)
    mse, rmse, mape = calculate_metrics(y_test, y_pred)
    print(f"Metrics for {title}:")
    print(f"MSE: {mse}")
    print(f"RMSE: {rmse}")
    print(f"MAPE: {mape}\n")

Conclusin KNN

Hasta ahora, el modelo K-Nearest Neighbors (KNN) ha demostrado ser el modelo de aprendizaje automtico que ofrece los mejores resultados en las series mensuales de pasajeros de autobs y metro de Madrid, superando al Decision Tree en cuanto a precisin y ajuste a los datos. Sin embargo, en las series trimestrales, el rendimiento del KNN no ha sido tan favorable, lo que sugiere que el modelo no captura adecuadamente las dinmicas estacionales o de largo plazo que caracterizan estos datos. A pesar de los resultados obtenidos con KNN en las series mensuales, los modelos ETS (Error, Trend, Seasonal) y Theta siguen siendo superiores en general, tanto en los resultados visuales como en las mtricas de error MSE, RMSE y MAPE. Esto indica que, aunque KNN es competitivo en algunos casos, los modelos ETS y Theta continan siendo las mejores opciones para predecir las series temporales de pasajeros, especialmente cuando se consideran tanto la precisin como la interpretabilidad de las predicciones.

En este caso, tambin hemos utilizado validacin cruzada para tener las mejores predicciones posibles.
