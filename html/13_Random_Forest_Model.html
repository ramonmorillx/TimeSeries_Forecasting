El modelo Random Forest es un algoritmo de aprendizaje automtico basado en un conjunto de rboles de decisin, diseado para mejorar la precisin y la robustez de las predicciones mediante el uso de mltiples rboles entrenados de manera aleatoria. Random Forest puede capturar tanto las relaciones no lineales como las interacciones complejas entre los diferentes factores que afectan la demanda de transporte. A travs de un proceso de "ensamblaje", donde se promedian las predicciones de mltiples rboles para reducir el sobreajuste y aumentar la generalizacin, este modelo puede ser particularmente til para manejar grandes cantidades de datos histricos con patrones estacionales, tendencias y eventos atpicos. Aunque Random Forest no es tan interpretable como un solo rbol de decisin, su capacidad para manejar datos ruidosos y su flexibilidad lo hacen adecuado para abordar la prediccin de series temporales en escenarios complejos como el de los pasajeros de transporte pblico.

Modelo Random Forest

Random Forest es un algoritmo de aprendizaje automtico basado en el concepto de ensamblaje, que combina mltiples rboles de decisin para mejorar la precisin de las predicciones. En lugar de construir un solo rbol de decisin, Random Forest genera una "bosque" de rboles, entrenando cada uno con una muestra aleatoria del conjunto de datos original, seleccionando aleatoriamente un subconjunto de caractersticas para cada divisin en cada rbol, lo que fomenta la diversidad entre ellos. Cada rbol realiza una prediccin independiente, y luego, para problemas de regresin, la prediccin final se obtiene promediando las salidas de todos los rboles; para clasificacin, se realiza mediante una votacin mayoritaria. Este enfoque ayuda a reducir el sobreajuste, mejorar la generalizacin del modelo y manejar de manera efectiva datos ruidosos y no lineales, lo que lo convierte en una herramienta robusta para una variedad de tareas de prediccin.

# Funcin para entrenar, predecir y calcular mtricas con RandomForest
def train_predict_plot_metrics(df, test_size, title):
    # Definir el nmero de observaciones para el test
    test_size = test_size
    
    # Crear las variables de entrenamiento y prueba
    X = np.arange(len(df)).reshape(-1, 1)
    y = df.values
    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=test_size, shuffle=False)
    
    # Definir el modelo y los parmetros para GridSearchCV
    model = RandomForestRegressor()
    param_distributions = {
        'n_estimators': np.random.randint(100, 500, 10),
        'max_depth': np.random.randint(10, 20, 10),
        'min_samples_split': np.random.randint(1, 10, 10),
        'min_samples_leaf': np.random.randint(1, 10, 10),
        'max_features': np.random.randint(1, 100, 10 )
    }
    
    # Aplicar GridSearchCV
    grid_search = RandomizedSearchCV(estimator=model, param_distributions=param_distributions, cv=5, scoring='r2')
    grid_search.fit(X_train, y_train)
    
    # Mejor modelo
    best_model = grid_search.best_estimator_
    
    # Predicciones
    y_train_pred = best_model.predict(X_train)
    y_test_pred = best_model.predict(X_test)
    
    # Calcular mtricas
    mse = mean_squared_error(y_test, y_test_pred)
    rmse = math.sqrt(mse)
    mape = mean_absolute_percentage_error(y_test, y_test_pred)
    
    # Graficar
    plt.figure(figsize=(14, 5))
    plt.plot(df.index, y, label='Actual', color='blue')
    plt.plot(df.index[:len(y_train)], y_train_pred, label='Train Prediction', color='green')
    plt.plot(df.index[len(y_train):], y_test_pred, label='Test Prediction', color='red')
    plt.axvline(df.index[len(y_train)], color='black', linestyle='--', label='Train/Test Split')
    plt.title(title)
    plt.xlabel('Date')
    plt.ylabel('Total')
    plt.legend()
    plt.show()
    
    # Imprimir el mejor score, los mejores parmetros y las mtricas
    print(f"Best R2 Score: {grid_search.best_score_}")
    print(f"Best Parameters: {grid_search.best_params_}")
    print(f"MSE: {mse}")
    print(f"RMSE: {rmse}")
    print(f"MAPE: {mape}")

Conclusin Random Forest

El entrenamiento de un modelo Random Forest implica un fuerte compromiso entre la precisin de las predicciones y los costes computacionales asociados. Para obtener buenos resultados, es crucial ajustar correctamente los hiperparmetros clave, como el nmero de rboles (n_estimators), la profundidad mxima de los rboles (max_depth), y el nmero de caractersticas consideradas en cada divisin (max_features). Sin embargo, este proceso puede suponerun gran coste en trminos de tiempo y recursos computacionales, especialmente cuando se trabaja con grandes conjuntos de datos o series temporales complejas, como la de los pasajeros de autobs y metro de Madrid. A medida que se incrementa el nmero de rboles o la profundidad, el modelo puede capturar mejor los patrones de los datos, pero tambin aumenta significativamente el tiempo de entrenamiento y el uso de memoria, lo que puede ser prohibitivo en entornos con recursos limitados.

Dado este escenario, el uso de GridSearchCV para realizar una bsqueda exhaustiva de hiperparmetros no resulta factible, ya que evala todas las combinaciones posibles, multiplicando exponencialmente el coste computacional. Como alternativa, he optado por usar RandomSearchCV, que realiza una bsqueda aleatoria dentro de un rango definido de hiperparmetros. Este enfoque reduce considerablemente el tiempo de cmputo al explorar solo una muestra aleatoria de combinaciones en lugar de todas, manteniendo una buena probabilidad de encontrar configuraciones cercanas al ptimo. Aunque RandomSearchCV no garantiza encontrar la mejor combinacin posible, su eficiencia y balance entre precisin y coste lo convierten en una eleccin ms prctica para el ajuste de modelos Random Forest en este contexto.
